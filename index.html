const

  1. days[9] = 44 會導致編譯錯誤，因為days被const保護，所以不能改變其val
    1. *pd = 244.2 和pd[2] = 22 都是錯誤的，pd的聲明為const
    2. rates未被const保護，仍可改變
    3. 雖pd是const,但仍可以pd++使pd指向rates[1]
  1. pointer 和 const的配合需要注意一下規則
    1. 可以把const或非const的數據賦給const的pointer
    2. 普通pointer只能接受非const數據

不能利用非const的參數修改const數據

所以const通常用於函數參數中，表面不會利用指針更改數據

指針和多維數組

zippol1.c

** 雙向間接 double indirection，使用*或在數組後使用[]皆能得到相應的val。

zippo[0] 是zippo[0][0] 的地址，*(zippo[0])表示存在zippo[0][0]的val，所以*zippo等於zippo[0]的val，但zippo[0]本身是一個int類型的地址，即&zippo[0][0]的地址。

*zippo 等於 &zippo[0][0]

所以， **zippo 等於*&zippo[0][0]

zippol2.c

  • int (*pz)[2] 不等於int * pz[2] ，因為[] 優於*

​ int (*pz)[2] 是指pz指向一個內有兩個int類型的數組

​ int * pz[2]是指pz是一個內有兩個int類型的數組，每個元素都指向int的ptr

  • 雖pz是一個ptr，但仍能如數組一樣，以pz[2][1]這種寫法表達
    • zippo[m][n] == *(*(zippo + m ) + n)
    • pz[m][n] == *(*(pz + m) + n)

*p2是指向int的指針，而ar2[0]是指向ar[0][0]的指針，所以同類型

第一個[]是空的，即表明pt是一個ptr

編譯器會把所有數組表示法轉成指針表示法。例如，把pt[1]轉為pt + 1

編譯器對pt+1求值，所以要求知道pt的大小

pt指向一個內有4個int類型的數組，一個int等於4bytes，所以pt指向的對象占16bytes。因此每當pt+1時，代表的是在pt的地址上加上16bytes。如果第2個[] 為空，編譯器則無法處理

基本

  • () 左至右

  • + - 右至左 e.g. -6.4 +7

  • * / 左至右

  • + - 左至右 e.g. 80 + 9

  • = 右至左 e.g. s = 9

其他

  • sizeof 返回運算對象大小（上一篇有提及）

    size_t 無符號整數類型

  • % 用於求餘數（只限整數）

  • ++ ／ –

    count++ 利用count的val，後+1

    ++count 先+1，後用count的val

優先

x*y++ 等於 (x)*(y++)

因為++/-- 只能影響一個val

  • 無符號轉整數，額外的位會被忽略
  • 有符號轉整數，因實現而異
  • 整型轉浮點數，行為未定義

C basic ch3(數據類型)

數據類型

位、字節、和字

  • bit (位)
    • 最小的存儲空間
    • 存 0 ／ 1
  • byte(字節)
    • 1 byte == 8 bit
    • 8 byte 有256個0、1組合， 即是2^8
    • 通過2進制，可以表示0-255/一組字符char

整數 int

  • short int(16bit) & int(16/32bit)

    • -32767 - 32767

  • long int / long(32bit)

    • -2147483647 - 2147483647

  • long long int / long long (64bit)

  • C90: unsigned int / unsigned

    • 只適用非負數
    • range： 0 - 65535

  • C90: unsigned short int / unsigned short

  • C99: unsigned long long int / unsigned long long

  • 關鍵字 signed， 強調使用有符號

整數溢出

如果整數超越取值範圍，會重新從起點開始

浮點數float

  • float
  • double
  • long double

字符char

C lang 把1byte定義為char類型

注意char類型需要用單引號，因為編譯器用單引號來分別char和string

_Bool類型

true false

  • 0 代表 false
  • 1 代表 true

類型大小

sizeof 內置運算符， 需配合%zd / %u / %lu

  • int = 4 bytes
  • char = 1 bytes
  • long / long long = 8 bytes
  • double = 8 bytes
  • lond doube = 16 bytes
